Variables & Data Types

A variable is a named container for a value. Think of it like a labelled box — you give it a name, put something inside, and can use or change that thing later by referring to the name.

let vs const vs var

let creates a variable you can reassign. const creates a variable that cannot be reassigned. var is the old way — it has confusing scoping rules and is considered legacy. Never use it.

let score = 0;          // can be changed later
score = 10;             // ✅ fine — we used let

const name = 'Darian';  // cannot be changed
// name = 'Bob';         // ❌ TypeError: Assignment to constant variable

// Rule: use const by default. Switch to let only when the value must change.
// Loop counters (i), accumulators (result), score trackers → let
// Everything else → const

const with objects and arrays: const prevents reassignment but NOT mutation. You can't replace the array entirely, but you CAN push items into it. This catches many beginners off guard.

const arr = [1, 2, 3];
arr.push(4);           // ✅ allowed — we're modifying, not replacing
// arr = [5, 6, 7];   // ❌ not allowed — that's reassignment

The 6 Primitive Data Types

Every value in JavaScript is one of these types. Knowing the type of a value tells you what you can do with it.

String'hello', "world"Text — always wrapped in quotes (single, double, or backtick)
Number42, 3.14, -7Any number — JS has no separate int/float distinction
Booleantrue, falseOnly two possible values — used for conditions

typeof — Checking a Value's Type

When you're confused about what type a value is, use typeof to check.

typeof 'hello'     // 'string'
typeof 42          // 'number'
typeof true        // 'boolean'
typeof undefined   // 'undefined'
typeof null        // 'object'  ← famous JS bug, null is NOT an object
typeof [1,2,3]     // 'object'  ← arrays are also objects in JS
typeof function(){} // 'function'

Type Coercion — JavaScript's Most Confusing Behaviour

JavaScript automatically converts types in certain situations. This causes bugs if you're not careful.

'5' + 3     // '53'  — JS converts 3 to string and concatenates
'5' - 3     // 2     — JS converts '5' to number and subtracts
'5' * 2     // 10    — same, converts to number
true + 1    // 2     — true becomes 1
false + 1   // 1     — false becomes 0

// This is why you always use === not ==
5 == '5'    // true  — == does coercion, dangerous
5 === '5'   // false — === checks type AND value, safe

Naming Variables

  • Always camelCase: totalScore, isFinished, userName
  • Name what it contains, not what you do with it — userAge not myVariable
  • Booleans should read like yes/no questions: isWinner, hasError, canSubmit
  • Constants that never change anywhere use SCREAMING_SNAKE_CASE: MAX_GRID_SIZE

Operators

Arithmetic Operators

10 + 3   // 13 — addition
10 - 3   // 7  — subtraction
10 * 3   // 30 — multiplication
10 / 3   // 3.333... — division (always returns decimal if needed)
10 % 3   // 1  — modulo: remainder after division
10 ** 2  // 100 — exponentiation (10 to the power of 2)

// Shorthand versions (all do the same thing, just shorter)
x += 5   // same as x = x + 5
x -= 5   // same as x = x - 5
x *= 2   // same as x = x * 2
x++      // same as x = x + 1 (increment by 1)
x--      // same as x = x - 1 (decrement by 1)

Modulo % — The Most Important One

Returns the remainder after division. You use this constantly — for checking if something is even/odd, for leap years, for cycling through values.

4 % 2    // 0 — even numbers always have remainder 0 when divided by 2
7 % 2    // 1 — odd numbers always have remainder 1
10 % 3   // 1 — 10 divided by 3 is 3 remainder 1
100 % 400 // 100 — used in leap year check

// Check if number is even:
if (num % 2 === 0) { console.log('even'); }

// Check divisibility (leap year logic):
if (year % 400 === 0) { return true; }   // divisible by 400
if (year % 100 === 0) { return false; }  // divisible by 100 but not 400
if (year % 4 === 0) { return true; }     // divisible by 4 but not 100

Comparison Operators

These always return true or false. They're used inside if statements.

5 === 5    // true  — strict equal (checks value AND type)
5 !== 3    // true  — strict not equal
5 > 3      // true  — greater than
5 < 3      // false — less than
5 >= 5     // true  — greater than or equal to
5 <= 4     // false — less than or equal to

// NEVER use == or != (loose equality)
5 == '5'   // true  — dangerous, converts types automatically
5 === '5'  // false — safe, different types = not equal

Logical Operators

Used to combine multiple conditions.

true && true    // true  — AND: both must be true
true && false   // false — AND: one is false so result is false
true || false   // true  — OR: at least one is true
false || false  // false — OR: both false so result is false
!true           // false — NOT: reverses the boolean
!false          // true

// Real usage:
if (age >= 18 && hasId) { console.log('allowed'); }
if (isAdmin || isModerator) { console.log('access granted'); }
if (!isLoggedIn) { console.log('please log in'); }

Conditionals

Conditionals let your code make decisions. The program checks a condition and runs different code depending on whether it's true or false.

if / else if / else

The most common way to make decisions. Only one block runs — whichever condition is true first.

if (score >= 90) {
  console.log('A');
} else if (score >= 80) {
  console.log('B');
} else if (score >= 70) {
  console.log('C');
} else {
  console.log('F');
}

Ternary Operator

const label = age >= 18 ? 'adult' : 'minor';
const result = humanScore > computerScore ? 'You win!' : 'Computer wins!';

Switch Statement

switch (day) {
  case 'Monday': console.log('Start of week'); break;
  case 'Friday': console.log('End of week'); break;
  case 'Saturday':
  case 'Sunday': console.log('Weekend'); break;
  default: console.log('Midweek');
}

Truthy and Falsy Values

// Falsy: false, 0, '', null, undefined, NaN
// Everything else is truthy
if (userName) { console.log('Hello ' + userName); }
if (!errors) { console.log('No errors found'); }

Functions

A function is a reusable block of code. You define it once and can run it as many times as you want.

Three Ways to Write a Function

// 1. Function Declaration — hoisted
function add(a, b) { return a + b; }

// 2. Function Expression — NOT hoisted
const add = function(a, b) { return a + b; };

// 3. Arrow Function — modern standard
const add = (a, b) => a + b;
const add = (a, b) => { return a + b; };

Parameters, Return Values, Defaults

function greet(name = 'stranger') {
  return 'Hello ' + name;
}
greet('Alice')  // 'Hello Alice'
greet()         // 'Hello stranger'

Callbacks

button.addEventListener('click', () => { console.log('clicked'); });

Scope

Scope determines where a variable is accessible. let and const are block-scoped. var is function-scoped and leaks — never use it.

const globalVar = 'everywhere';
function myFunction() {
  const localVar = 'only here';
  console.log(globalVar);  // ✅
}
console.log(localVar);    // ❌ ReferenceError

if (true) { let blockVar = 'inside'; }
console.log(blockVar);    // ❌ ReferenceError — block-scoped

Loops

for Loop

for (let i = 0; i < 5; i++) { console.log(i); }
// Prints: 0, 1, 2, 3, 4

forEach

const fruits = ['apple', 'banana', 'cherry'];
fruits.forEach(fruit => console.log(fruit));
fruits.forEach((fruit, index) => console.log(index, fruit));

Nested Loop (used in Etch-a-Sketch)

for (let row = 0; row < 3; row++) {
  for (let col = 0; col < 3; col++) {
    console.log(`row ${row}, col ${col}`);
  }
}

while, break, continue

let i = 0;
while (i < 5) { console.log(i); i++; }

for (let i = 0; i < 10; i++) {
  if (i === 5) break;      // exit loop
  if (i === 2) continue;   // skip iteration
}

The Accumulator Pattern

The most important pattern in beginner programming. Create a variable before the loop, build it up inside, return after.

The Universal Shape

let result = startingValue;    // '', 0, or []
for (let i = 0; i < something; i++) {
  result = result + piece;
}
return result;
TaskStart withWhy
Build a stringlet result = ''Adding text to empty string gives you that text
Sum numberslet sum = 0Adding number to 0 gives you that number
Collect into arraylet result = []Push items into the empty array
Count somethinglet count = 0Add 1 each time condition matches
Multiply numberslet result = 1Multiplying by 1 is a safe start

Reversing a String

// Loop:
function reverseString(str) {
  let result = '';
  for (let i = str.length - 1; i >= 0; i--) result += str[i];
  return result;
}
// One-liner:
const reverseString = str => str.split('').reverse().join('');

Array Methods — map, filter, reduce

.map() — Transform Every Item

const doubled = [1,2,3].map(n => n * 2);  // [2,4,6]
const names = users.map(user => user.name);

.filter() — Keep Matching Items

const evens = [1,2,3,4].filter(n => n % 2 === 0);  // [2,4]
// removeFromArray uses this:
return array.filter(item => !args.includes(item));

.reduce() — Boil to One Value

const sum = [1,2,3,4,5].reduce((acc, n) => acc + n, 0);  // 15

Chaining

const result = numbers
  .filter(n => n % 2 === 0)
  .map(n => n * 3)
  .reduce((acc, n) => acc + n, 0);

Spread & Rest

// Rest — collect remaining args into array:
function removeFromArray(array, ...args) {
  return array.filter(item => !args.includes(item));
}

// Spread — expand array/object:
Math.max(...[1,5,3]);         // 5
const copy = [...original];   // copy array
const merged = [...a, ...b];  // merge arrays
const full = { ...person, ...extra }; // merge objects

DOM Manipulation

Always put your script tag at the bottom of body or use defer. JS that runs before HTML is parsed will find nothing.

Selecting Elements

document.querySelector('.class');

Problem-Solving Mindset

  1. Read the problem twice — restate in your own words before coding
  2. Write pseudocode — plain English steps as comments
  3. One tiny step at a time — don't write everything at once
  4. Test after every change — console.log after every new line
  5. Fix one failing test at a time
  6. Clean up after it works — refactor only once logic is correct
  7. Google the error — copy exact error + "JavaScript"

Being stuck is not failure — it's the work. Every developer at every level gets stuck. The skill is knowing how to get unstuck.
